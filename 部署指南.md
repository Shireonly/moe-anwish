## 一、Cloudflare DNS 配置（Dashboard 操作）

### 1.1 确认 DNS 记录
登录 [Cloudflare Dashboard](https://dash.cloudflare.com) → 选择域名 → DNS → Records：

| 类型 | 名称 | 内容 | 代理状态 |
|------|------|------|---------|
| CNAME | @ | arlwish.pages.dev | 已代理 (橙色云朵) |
| CNAME | www | arlwish.pages.dev | 已代理 (橙色云朵) |

### 1.2 SSL/TLS 设置
- SSL/TLS → 加密模式：**完全（严格）**
- 边缘证书 → 始终使用 HTTPS：**开启**
- 自动 HTTPS 重写：**开启**

### 1.3 缓存规则
- Caching → Cache Rules → 创建规则：
  - 当 URL 匹配 `*.html` 或 `/` → 缓存 TTL: 1小时
  - 当 URL 匹配 `*.css` `*.js` `*.svg` `*.png` `*.ico` → 缓存 TTL: 1年
  - 当 URL 匹配 `/ws` → 绕过缓存

### 1.4 速度优化
- Speed → Optimization → 
  - Auto Minify: JS/CSS/HTML 全开
  - Brotli: **开启**
  - Early Hints: **开启**
  - Rocket Loader: **关闭**（WebSocket 应用不兼容）

### 1.5 安全设置
- Security → WAF → 
  - 如果误拦截中国 IP，添加规则放行
  - 确认 Bot Fight Mode 不会拦截正常 WebSocket 连接

## 二、现在你需要做的事

### 第一步：部署最新代码（3 选 1）

**方案 A — 直接运行 Python 脚本（最快）**:
```bash
cd chatroom
python _deploy_v3.py
```

**方案 B — 安装 Wrangler CLI（推荐）**:
```bash
npm install -g wrangler
wrangler login
wrangler pages deploy deploy/ --project-name=arlwish
```

**方案 C — GitHub 自动部署（一劳永逸）**:
1. 在项目目录 `git init && git add . && git commit -m "milimiki v2"`
2. Push 到 GitHub
3. Cloudflare Dashboard → Workers & Pages → arlwish → 设置 → Build & Deploy → 连接 GitHub 仓库
4. 设置构建输出目录为 `deploy/`，构建命令留空
5. 以后每次 `git push` 自动部署

### 第二步：Cloudflare Dashboard 设置

1. **DNS**: 确认 `CNAME milimiki.com → arlwish.pages.dev`（橙色云朵开启）
2. **SSL/TLS**: 加密模式 → 完全（严格）
3. **Speed → Optimization**: Auto Minify 全开、Brotli 开启、Early Hints 开启、Rocket Loader 关闭
4. **Caching → Cache Rules**: 给 `/ws` 路径设置绕过缓存

### 第三步：启动/确认 Python 后端

后端需要运行在服务器上，端口 8766。建议用 `cloudflared tunnel` 暴露 WebSocket 端口。

### 第四步：清理密钥 ⚠️

部署成功后，`_deploy.py`、`_deploy_fresh.py`、`_deploy_v3.py` 里的 API Token 需要删除或替换。
在 Cloudflare Dashboard 撤销旧 Token，生成新的，然后把这些脚本加入 `.gitignore`。
